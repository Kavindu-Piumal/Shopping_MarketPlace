üêõ ISSUE IDENTIFIED

Problem: On PC view home page, after clicking "Add to Cart", only the - button was visible in the quantity selector. The + button was cut off.
Root Cause: CSS media query was applying to all screens below 1024px (including desktop), forcing 96px width and 32px button sizes, but this conflicted with desktop layout needs.
Context: Product detail pages worked fine, but home page product cards had the issue.

‚úÖ FIX IMPLEMENTED

Solution: Changed CSS media query from @media (max-width: 1023px) to @media (max-width: 640px)
Impact: Now the forced sizing only applies to mobile devices, allowing desktop to use natural Tailwind CSS sizing
Technical Details:
  • Desktop: Uses component's w-24 (96px) naturally
  • Mobile: Still gets forced 96px for consistency
  • All three buttons (-, qty, +) now properly visible
